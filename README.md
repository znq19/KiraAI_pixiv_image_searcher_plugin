# Pixiv 图片搜索插件

一个适用于 [KiraAI](https://github.com/) 的插件，用于从 Pixiv 搜索高人气图片并直接发送到 QQ 聊天。

## 功能

- 😎 高度节省token和快速步骤但又能让LLM明白自己发过图片
- 🔍 支持中文关键词，自动转换为 Pixiv 日文/英文 tag
- 🔥 按收藏数排序，支持设置最低收藏数门槛
- 🖼️ 默认以**合并转发**形式发送图片，避免刷屏
- 🗂️ 支持下载**原图**或**预览图**
- 🚫 内置 R-18 过滤和标签排除
- 🔄 发送过的图片默认 24 小时内会去重
- 🧹 自动清理本地图片缓存

## 适用场景

- 在群里让机器人搜图
- 私聊让机器人按关键词发图
- 快速分享 Pixiv 高人气插画

## 安装

1. 把插件文件夹放到 KiraAI 的 `data/plugins/` 目录下
2. 文件夹名建议保持为 `pixiv_image_searcher`
3. 重启 KiraAI，或在 WebUI 中重载插件

```text
KiraAI/
└── data/
    └── plugins/
        └── pixiv_image_searcher/
            ├── main.py
            ├── manifest.json
            ├── schema.json
            └── requirements.txt
```

## 配置

在 KiraAI WebUI 中找到本插件，填写以下配置：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| **Refresh Token** | 空 | Pixiv 登录凭证，必填 |
| **HTTP 代理** | 空 | 如 `http://127.0.0.1:7890`，国内网络建议填写 |
| **最大返回张数** | 5 | 单次最多发几张，1~20 |
| **最低收藏数阈值** | 100 | 只发收藏数大于等于该值的作品 |
| **排除标签** | AI, R-18, 裸足 | 命中任一标签的作品会被过滤 |
| **私聊允许发送 R-18** | 关闭 | 开启后私聊中允许 R-18（x_restrict=1），并忽略排除标签中的 R-18 |
| **私聊允许发送 R-18G** | 关闭 | 开启后私聊中允许 R-18G（x_restrict=2），并忽略排除标签中的 R-18G |
| **R-18 会话白名单** | 空 | 允许发送 R-18 的会话 ID 列表，格式 `qq:dm:123456` / `qq:gm:789012` |
| **R-18G 会话白名单** | 空 | 允许发送 R-18G 的会话 ID 列表，格式同上 |
| **最大搜索页数** | 100 | 搜索时翻页上限 |
| **关键词映射表** | 见配置 | 中文关键词转 Pixiv tag |
| **去重过期时间** | 24 小时 | 0 表示永久去重 |
| **图片缓存目录** | `files/pixiv` | 相对 `data/` 目录 |
| **默认下载原图** | 关闭 | 开启后默认发原图 |
| **以合并转发形式发送** | 开启 | 关闭后逐条发送图片 |
| **缓存文件数量上限** | 50 | 超过会触发清理 |
| **每次清理旧文件数量** | 20 | 每次清理最古老的文件数 |

## 如何获取 Refresh Token

本插件需要 Pixiv 的 `refresh_token` 才能登录。

### 方式一：使用本仓库附带的工具

本仓库附带一个独立的 [pixiv-token-tool](https://github.com/znq19/pixiv-token-tool) 工具，双击运行即可：

```text
pixiv-token-tool/
├── run.bat              # Windows 一键运行
├── get_pixiv_token.py   # 主程序
├── requirements.txt
└── README.md
```

1. 双击 `run.bat`
2. 在打开的浏览器中登录 Pixiv
3. 登录成功后，`refresh_token.txt` 会生成在同一目录
4. 把 `refresh_token.txt` 里的内容复制到插件配置中

### 方式二：使用 gppt 命令行工具

如果你已经会用 Python：

```bash
pip install gppt
gppt login -j
```

按提示在浏览器中登录 Pixiv，获取 `refresh_token`。

## 使用方法

配置好 Refresh Token 后，重启插件。然后在聊天中发送类似消息：

```text
发几张黑丝图
来张原神图
给我看看泳装
发3张白丝，要原图
```

机器人会自动调用插件，搜索并发送图片。

### 常用说法

| 你说 | 机器人会 |
|------|---------|
| "来张黑丝" | 搜索 `黒タイツ`，发预览图 |
| "发3张原神图" | 搜索 `原神`，发3张 |
| "这张发原图" / "发原图 12345678" | 按作品 ID 发原图 |
| "再来点" | 继续搜索并发送（24h 后会重复） |

## 常见问题

### 1. 插件初始化失败 / Token 验证失败

- 检查 Refresh Token 是否填写正确
- Token 可能已过期，请重新获取
- 检查代理是否配置正确

### 2. 搜索无结果

- 降低"最低收藏数阈值"，例如从 10000 改为 1000
- 增加"最大搜索页数"
- 检查网络/代理是否能正常访问 Pixiv

### 3. 图片发不出来

- 检查代理是否能访问 `i.pximg.net`
- 尝试关闭"默认下载原图"，先发送预览图测试
- 查看 KiraAI 日志中的具体错误

### 4. 图片发送成功但机器人又开始识别自己发的图

- 本插件已给发送的图片设置 `caption=""`，理论上不会触发 VLM 描述
- 如果仍触发，可能是 QQ 消息回显导致，可在系统侧配置忽略机器人自己发送的图片

### 5. 合并转发消息不显示

- 某些 QQ 协议/版本对合并转发支持不完整
- 可关闭"以合并转发形式发送"，改为逐条发送

### 6. 插件如何做到屏蔽R-18 内容的？

插件有两层 R-18/R-18G 过滤：

1. **`x_restrict` 字段过滤**：Pixiv API 返回的 `x_restrict` 字段，`0` 是全年龄，`1` 是 R-18，`2` 是 R-18G。
2. **标签过滤**：`exclude_tags` 默认包含 `"R-18"`，命中该标签的作品会被过滤。

从 v1.3.0 开始，插件新增**会话白名单**机制，规则如下：

| 场景 | R-18 / R-18G 是否可发送 |
|------|------------------------|
| 私聊 + 对应开关关闭 | ❌ 不允许 |
| 私聊 + 开关开启 + 白名单为空 | ✅ 所有私聊都允许 |
| 私聊 + 开关开启 + 白名单非空 | ✅ 仅白名单中的私聊允许 |
| 群聊 + 白名单为空 | ❌ 所有群聊都不允许 |
| 群聊 + 白名单非空 | ✅ 仅白名单中的群聊允许 |

白名单格式为完整会话 ID，例如 `qq:dm:123456`、`qq:gm:789012`。R-18 与 R-18G 分别有独立的白名单。

因此，只把 `exclude_tags` 里的 `"R-18"` 去掉**仍然发不了 R-18/R-18G**，必须同时满足 `x_restrict` 和白名单规则。
